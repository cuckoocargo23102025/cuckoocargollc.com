02
CÀI ĐẶT CÔNG CỤ
Cài 1 lần duy nhất — sau đó dùng mãi mãi
Thứ tự quan trọng! Cài theo đúng thứ tự bên dưới để tránh lỗi. Mỗi bước tôi đã ghi link download và lệnh cài đặt cụ thể.
1
Homebrew (chỉ Mac) — Trình quản lý phần mềm
Homebrew giúp cài các phần mềm lập trình trên Mac dễ dàng hơn nhiều.
Mở Terminal (Command + Space → gõ "Terminal" → Enter) rồi paste lệnh sau:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Gõ mật khẩu Mac khi được hỏi. Chờ khoảng 3–5 phút.
2
Node.js — "Máy chạy" JavaScript
Node.js cho phép chạy JavaScript (ngôn ngữ lập trình web) trên máy tính của bạn.
# Cài Node.js phiên bản ổn định (LTS) brew install node@20 # Kiểm tra đã cài thành công chưa node --version # Phải hiện: v20.x.x npm --version # Phải hiện: 10.x.x
3
Git — Hệ thống lưu lịch sử code
Git như "Ctrl+Z không giới hạn" cho code — bạn có thể quay lại bất kỳ phiên bản nào trong quá khứ.
brew install git # Đặt tên và email (dùng email GitHub của bạn) git config --global user.name "Tuan Vo" git config --global user.email "vohoangtuan0601@gmail.com"
4
VS Code — Trình soạn thảo code (Editor)
VS Code là nơi bạn xem và chỉnh sửa code. Dù Claude Code tự viết, bạn vẫn cần VS Code để xem kết quả.
Download: code.visualstudio.com → Chọn "Download for Mac" → Cài như app thường.
Extensions cần cài trong VS Code:
# Mở VS Code → nhấn Ctrl+P (hoặc Cmd+P trên Mac) → paste từng dòng ext install bradlc.vscode-tailwindcss ext install esbenp.prettier-vscode ext install dbaeumer.vscode-eslint ext install Prisma.prisma
5
Claude Code — AI viết code cho bạn
Đây là công cụ quan trọng nhất! Claude Code chạy trong Terminal và sẽ viết toàn bộ code theo lệnh của bạn bằng tiếng Việt.
# Cài Claude Code qua npm npm install -g @anthropic-ai/claude-code # Kiểm tra đã cài chưa claude --version
Cần API Key của Anthropic. Đăng ký tại console.anthropic.com → Tạo API Key → Chạy claude login và paste key vào.
6
MySQL Workbench — Xem dữ liệu trực quan
MySQL Workbench giúp bạn xem bảng dữ liệu trực quan như Excel — không cần gõ lệnh phức tạp.
Download: dev.mysql.com/downloads/workbench → Chọn macOS → Download.
7
Docker Desktop — Chạy database trên máy local
Docker tạo ra một "máy tính ảo" nhỏ trong máy bạn để chạy MySQL và Redis mà không cần cài phức tạp.
Download: docker.com/products/docker-desktop → Chọn "Download for Mac".
# Kiểm tra Docker đã hoạt động docker --version # Phải hiện: Docker version 24.x.x

04
KHỞI TẠO DỰ ÁN
Tạo cấu trúc thư mục và kết nối GitHub — làm 1 lần duy nhất
Tạo một thư mục chính cho toàn bộ dự án. Mọi code, file, database đều sẽ nằm trong thư mục này. Đề xuất: ~/Desktop/cuckoo-cargo/
1
Tạo thư mục dự án
# Mở Terminal → paste từng dòng cd ~/Desktop mkdir cuckoo-cargo cd cuckoo-cargo # Tạo 2 thư mục con: frontend và backend mkdir frontend backend ls # Kiểm tra: phải thấy "frontend" và "backend"
2
Tạo Repository trên GitHub
Repository = "kho chứa code" trên GitHub. Mỗi khi bạn thay đổi code, GitHub lưu lại lịch sử.
1. Mở github.com → Đăng nhập → Nhấn nút "New" (màu xanh)
2. Repository name: cuckoo-cargo
3. Chọn Private (không public code)
4. Nhấn "Create repository"
# Kết nối thư mục local với GitHub cd ~/Desktop/cuckoo-cargo git init git remote add origin https://github.com/TÊN_GITHUB_CỦA_BẠN/cuckoo-cargo.git
3
Tạo dự án Next.js (Frontend) — Claude Code làm hộ
Mở Terminal, điều hướng vào thư mục frontend, rồi khởi động Claude Code:
# Vào thư mục frontend cd ~/Desktop/cuckoo-cargo/frontend # Khởi động Claude Code claude # Rồi gõ lệnh cho Claude (bằng tiếng Việt): > Hãy tạo project Next.js 14 với App Router, Tailwind CSS. Tên project là cuckoo-cargo-frontend. Cài thêm: next-intl (để hỗ trợ 2 ngôn ngữ vi/en). Tạo cấu trúc thư mục theo SOP của Cuckoo Cargo.
4
Tạo dự án Backend (Node.js + Express)
cd ~/Desktop/cuckoo-cargo/backend claude > Tạo project Node.js với Express, Prisma ORM kết nối MySQL. Cài: express, prisma, @prisma/client, cors, helmet, express-rate-limit, bcryptjs, jsonwebtoken, multer, dotenv. Tạo cấu trúc: src/routes, src/controllers, src/models, src/middleware, src/services.
5
Tạo file docker-compose.yml — Chạy MySQL và Redis local
cd ~/Desktop/cuckoo-cargo claude > Tạo file docker-compose.yml để chạy: - MySQL 8.0 với database tên "cuckoo_cargo", user "cargo_user", password "CargoPass2026!" - Redis 7 để cache - Expose MySQL port 3306, Redis port 6379
# Khởi động MySQL và Redis (mỗi lần bắt đầu làm việc) docker-compose up -d # Kiểm tra đang chạy docker ps
Sau bước này bạn có: Thư mục dự án đầy đủ + code frontend + code backend + MySQL đang chạy trên máy. Đây là "môi trường phát triển" (development environment) của bạn.

SETUP DATABASE (MySQL)
Tạo các bảng dữ liệu theo SOP — Claude Code làm hết
1
Tạo file .env — Cấu hình kết nối Database
File .env chứa thông tin nhạy cảm (mật khẩu, API keys). TUYỆT ĐỐI không chia sẻ file này với ai.
# File: backend/.env DATABASE_URL="mysql://cargo_user:CargoPass2026!@localhost:3306/cuckoo_cargo" REDIS_URL="redis://localhost:6379" JWT_SECRET="thay-bang-chuoi-ngau-nhien-dai-64-ky-tu" CLOUDINARY_CLOUD_NAME="tên-cloud-của-bạn" CLOUDINARY_API_KEY="api-key-từ-cloudinary" CLOUDINARY_API_SECRET="api-secret-từ-cloudinary" SENDGRID_API_KEY="SG.xxxxxxxxxxxxx" PORT=3001
2
Tạo Database Schema — Claude Code viết hết
Nói với Claude Code tạo toàn bộ database theo SOP:
cd ~/Desktop/cuckoo-cargo/backend claude > Tạo Prisma schema cho MySQL với đầy đủ các bảng theo SOP Cuckoo Cargo: - users (id, uuid, full_name, email, phone_us, phone_vn, password_hash, role, lang_pref, created_at) - orders (id, order_code CC-2026-VN-XXXXX, user_id, service_type, status, weight_lbs, dimensions, price_usd, payment_status, payment_method, note) - recipients (id, order_id, full_name, phone, province, district, ward, address_detail, is_hcm) - tracking_events (id, order_id, status, location, note_vi, note_en, photo_url, created_by, created_at) - pricing_rules (id, service_type, from_state, to_province, price_per_lb, surcharge_province, heavy_fee, base_price, est_days_min, est_days_max, is_active) - products (id, name_vi, name_en, desc_vi, desc_en, price_usd, stock_qty, images JSON, weight_lbs, category, is_active) Sau đó chạy migration để tạo các bảng trong MySQL local.
Claude sẽ tự động: Viết file schema.prisma → Chạy npx prisma migrate dev → Tạo đầy đủ các bảng trong MySQL → Bạn chỉ cần xác nhận.
3
Nhập dữ liệu ban đầu (Seed Data)
> Tạo seed data cho database: - Bảng pricing_rules: giá US→VN là $X/lbs, phụ thu tỉnh $0.99/lbs, phí nặng >50lbs $25 - Tài khoản admin: email admin@cuckoocargo.com, password Admin2026! - 3 sản phẩm mẫu: Ensure Original, Dầu gió Eagle, Glucosamine Chạy seed file luôn sau khi tạo xong.

QUY TRÌNH CHO MỖI TÍNH NĂNG MỚI
Bước 1
Mô tả tính năng cho Claude
Nói Claude biết bạn muốn làm gì. Càng chi tiết càng tốt — màu sắc, layout, dữ liệu nào cần hiển thị, validation gì.
Bước 2
Claude viết code và giải thích
Claude sẽ tạo/sửa các file cần thiết, giải thích từng bước. Nếu không hiểu chỗ nào, cứ hỏi thêm.
Bước 3
Mở trình duyệt kiểm tra
Xem kết quả tại localhost:3000. Thử click, điền form, kiểm tra trên mobile (F12 → Toggle device toolbar).
Bước 4
Phản hồi cho Claude
Nếu có lỗi hoặc muốn thay đổi, mô tả cho Claude: "Nút này màu sai", "Form bị lỗi khi nhập...", "Thêm animation vào đây". Claude sẽ sửa ngay.
Bước 5
Lưu code lên GitHub
Mỗi khi hoàn thành 1 tính năng, lưu lại để không mất code.
git add . git commit -m "feat: thêm trang tracking vận đơn" git push origin main

CÁCH RA LỆNH CHO AI
Các mẫu câu prompt hiệu quả — copy & dùng ngay
Nguyên tắc vàng: Prompt tốt = Ai làm gì + Trông như thế nào + Dữ liệu nào + Điều kiện gì. Đừng ngắn quá — càng chi tiết Claude làm càng đúng ý bạn.
Tạo trang / giao diện mới
Bạn gõ vào Claude:
Tạo trang /dich-vu/gui-hang-my-ve-viet-nam với: - Hero section: nền đen #0F0F10, tiêu đề Bebas Neue "GỬI HÀNG MỸ → VIỆT NAM", màu cam #F47B20 - Section giải thích quy trình 5 bước (icon → text) - Bảng giá hiển thị từ API /pricing/rules - FAQ accordion (5 câu hỏi thường gặp) - CTA cuối trang: nút "ĐẶT ĐƠN NGAY" màu cam - Responsive hoàn toàn trên mobile - SEO meta tags đầy đủ, keyword "gửi hàng Mỹ về Việt Nam Los Angeles"
Claude sẽ: Tạo file trang mới → Viết component → Gọi API → Responsive CSS → Meta tags
Tạo API / Backend
Bạn gõ vào Claude:
Tạo API endpoint POST /api/orders để khách hàng đặt đơn hàng: - Nhận dữ liệu: service_type, weight_lbs, dimensions, thông tin người gửi, thông tin người nhận - Validate: weight phải > 0, email đúng format, SĐT VN bắt đầu bằng 0 - Tự động tính giá theo bảng pricing_rules trong database - Tự động tạo order_code theo format CC-2026-VN-XXXXX (số tăng dần) - Lưu vào database MySQL - Gửi email xác nhận đến khách qua SendGrid - Trả về JSON: { success: true, order_code: "...", price_usd: 45.50 }
Claude sẽ: Viết controller → Validation → Logic tính giá → Lưu DB → Gửi email → Test với Postman
Sửa lỗi
Bạn gõ vào Claude:
Trang /theo-doi bị lỗi: khi tôi nhập mã vận đơn CC-2026-VN-00421 và nhấn nút TRACK thì console hiện lỗi: "TypeError: Cannot read properties of undefined (reading 'status')" Đây là lỗi gì và fix như thế nào?
Claude sẽ: Phân tích lỗi → Tìm file gây lỗi → Sửa → Giải thích tại sao lỗi
Chỉnh giao diện
Bạn gõ vào Claude:
Trong trang chủ (index.html hoặc page.tsx): - Nút "Đặt đơn ngay" đang màu xanh, đổi thành màu cam #F47B20 với gradient sang #D4650E - Thêm hiệu ứng hover: nút nổi lên 3px khi di chuột vào - Font tiêu đề đang là Arial, đổi thành Bebas Neue letter-spacing 3px - Trên mobile (< 768px): 2 nút CTA xếp theo chiều dọc thay vì ngang
Claude sẽ: Sửa đúng các file CSS/component → Không động vào phần khác
Mẹo quan trọng: Nếu Claude làm sai, đừng xóa và làm lại từ đầu. Chỉ cần nói: "Kết quả chưa đúng. Tôi muốn [mô tả lại]. Hãy sửa file [tên file] thôi, không được động vào các file khác."

ĐƯA WEBSITE LÊN MẠNG
Từ localhost → cuckoocargo.com — khi đã sẵn sàng launch
A
Deploy Frontend lên Vercel (Tự động)
Đây là bước đơn giản nhất — Vercel tự làm hết khi bạn kết nối GitHub:
1. Mở vercel.com → Đăng nhập bằng GitHub
2. Nhấn "New Project" → Chọn repo cuckoo-cargo
3. Root Directory: chọn frontend
4. Điền Environment Variables (copy từ file .env)
5. Nhấn Deploy → 2 phút sau website live!
Từ đây trở đi: Mỗi khi bạn git push, Vercel tự động build và deploy trong 2 phút. Không cần làm gì thêm!
B
Deploy Backend + Database lên Railway
1. Mở railway.app → Đăng nhập → New Project
2. Thêm service: MySQL → Railway tự tạo database
3. Thêm service: GitHub Repo → Chọn repo/backend
4. Điền Environment Variables → Nhấn Deploy
5. Railway cấp URL dạng: api-production.up.railway.app
C
Kết nối Domain cuckoocargo.com
# Cấu hình DNS tại Cloudflare (sau khi mua domain) # A Record: cuckoocargo.com → [Vercel IP] # CNAME Record: www → cname.vercel-dns.com api → [Railway domain] # SSL được Cloudflare + Vercel tự cấp — không cần làm gì thêm
Nhờ Claude Code giúp: Nói với Claude "Hướng dẫn tôi kết nối domain cuckoocargo.com với Vercel và Railway, tôi đang dùng Cloudflare DNS" — Claude sẽ hướng dẫn từng bước cụ thể.
